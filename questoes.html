ATIVIDADES TEÓRICAS COM FOCO EM WINDOWS E LINUX E SEGURANÇA CIBERNÉTICA

1. Definindo os Paradigmas: Código Aberto vs. Fechado

Sistema Operacional de Código Aberto (Open Source)

Definição: Um sistema operacional cujo código-fonte é disponibilizado publicamente, permitindo que qualquer pessoa veja, modifique e redistribua o software.

Vantagens:
  1. Customização: Permite alterações no sistema conforme necessidade.
  2. Gratuito: Não requer licença paga na maioria dos casos.
Desvantagens:
  1. Suporte limitado: Depende da comunidade ou empresas terceirizadas.
  2. Curva de aprendizado: Pode ser complexo para usuários iniciantes.

Exemplos: Linux (Ubuntu, Fedora), FreeBSD, Android (baseado em Linux)

Sistema Operacional de Código Fechado (Proprietário)

Definição: Um sistema operacional cujo código-fonte não é acessível ao público, sendo controlado por uma empresa.

Vantagens:
  1. Suporte oficial: Atendimento direto da empresa desenvolvedora.
  2. Facilidade de uso: Interface mais amigável para usuários comuns.
Desvantagens:
  1. Custo: Geralmente requer compra de licença.
  2. Restrições de personalização: Não permite modificações profundas.

Exemplos: Windows, macOS, iOS

2. Análise Comparativa de Instalação

Aspecto Windows (Código Fechado) Ubuntu (Código Aberto)
Licenciamento Requer chave de ativação. Gratuito, sem licença.
Personalização Poucas opções de customização. Permite escolher particionamento e pacotes.
Drivers Geralmente instalados automaticamente. Pode exigir instalação manual de drivers proprietários.

3. Debate: Modo Texto vs. Interface Gráfica

Modo Texto (CLI - Terminal)

Vantagens:
  • Mais rápido para tarefas repetitivas.
  • Consome menos recursos do sistema.
  • Permite automação via scripts.
Desvantagens:
  • Curva de aprendizado mais acentuada.
  • Menos intuitivo para usuários comuns.

Cenários indicados: Administração de servidores, tarefas de rede, automação.

Interface Gráfica (GUI)

Vantagens:
  • Facilidade de uso (arrastar, clicar).
  • Visualização intuitiva de arquivos e programas.
Desvantagens:
  • Consome mais memória e CPU.
  • Menos eficiente para tarefas avançadas.

Cenários indicados: Uso doméstico, edição de imagens/vídeos, navegação web.

5. Modelos de Usuários e Permissões

Gerenciamento de Usuários

  • Windows (NTFS):
    • Usa contas de usuário (Admin, Usuário Padrão).
    • Permissões baseadas em ACLs (Listas de Controle de Acesso).
  • Linux (rwx):
    • Usuários e grupos (chmod, chown).
    • Permissões em formato rwx (leitura, escrita, execução).

Comparação

Sistema Permissões Exemplo
Windows Full Control, Modify, Read icacls arquivo.txt /grant User:F
Linux chmod 755 script.sh (rwxr-xr-x) sudo chown user:grupo arquivo

6. O Processo de Boot: Do Power-On ao Desktop

Etapas do Boot

  1. POST (Power-On Self-Test): Verificação de hardware.
  2. BIOS/UEFI: Inicializa o firmware e identifica o dispositivo de boot.
  3. Carregamento do Bootloader:
    • Windows: bootmgr → Carrega o kernel (ntoskrnl.exe).
    • Linux: GRUB → Carrega o kernel (vmlinuz).
  4. Inicialização do Kernel: Monta sistemas de arquivos e inicia serviços.
  5. Gerenciador de Login: Exibe tela de usuário (Windows: winlogon, Linux: GDM/LightDM).

Diferenças entre Código Aberto e Fechado

  • Windows: Processo mais automatizado, menos visível ao usuário.
  • Linux: Maior controle sobre o boot (GRUB permite selecionar kernels diferentes).

7. Redes: Compartilhamento, Mapeamento

a) Unidades e Pastas em Rede

Definição: São recursos compartilhados em uma rede local que permitem o acesso remoto a arquivos e diretórios.

Utilidade: Facilitam a colaboração entre usuários, centralizam armazenamento e permitem acesso a recursos sem necessidade de cópias locais.

b) Compartilhamento e Acesso

Termo Definição
Compartilhamento Processo de disponibilizar recursos (pastas, impressoras) para outros usuários na rede
Acesso Permissões concedidas para utilização dos recursos compartilhados (leitura, gravação, etc.)

c) Mapeamento de Unidades de Rede

Atribuição de uma letra de unidade (ex: Z:) a um recurso compartilhado na rede, permitindo acesso como se fosse um disco local.

Exemplo Prático:
  1. No Windows Explorer, clique com o botão direito em "Este Computador" → "Mapear unidade de rede"
  2. Digite o caminho da pasta compartilhada: \\servidor\documentos
  3. Selecione a letra da unidade (ex: Z:)
  4. Agora você pode acessar os arquivos remotos como se estivessem no seu computador

8. Fundamentos de Segurança Cibernética

a) Definição de Segurança Cibernética

Conjunto de práticas, tecnologias e processos para proteger sistemas, redes e dados contra ataques digitais, danos ou acesso não autorizado.

b) 5 Ameaças Cibernéticas Comuns

Ameaça Descrição
Malware Software malicioso (vírus, ransomware) que infecta sistemas
Phishing Tentativas de enganar usuários para obter informações sensíveis
Ataques DDoS Sobrecarga de servidores com tráfego falso
Exploits Aproveitamento de vulnerabilidades em softwares
Man-in-the-Middle Interceptação de comunicações entre duas partes

c) Vulnerabilidades em Sistemas

Fraquezas ou falhas em softwares, hardwares ou processos que podem ser exploradas por atacantes para comprometer a segurança.

d) Credenciais e sua Importância

Dados de autenticação (usuário/senha, certificados digitais) que comprovam identidade. Sua proteção é crucial pois são a "chave" para acessos privilegiados.

e) Engenharia Social

Técnicas psicológicas para manipular pessoas e obter informações confidenciais ou acesso a sistemas.

Exemplos:
  1. E-mail fingindo ser do suporte técnico pedindo sua senha
  2. Ligação telefônica se passando por colega de trabalho solicitando dados

9. Métodos de Proteção e Prevenção

Medidas para Usuários

  • Atualização regular de sistemas e aplicativos
  • Uso de autenticação multifator
  • Backups periódicos dos dados importantes
  • Criptografia de dados sensíveis
  • Não reutilizar senhas entre serviços

Medidas para Administradores

  • Implementação de políticas de segurança
  • Segmentação de redes
  • Monitoramento contínuo (SIEM)
  • Controle de privilégios (princípio do menor privilégio)
  • Testes de penetração regulares

Função do Firewall Nativo

Filtra tráfego de rede com base em regras pré-definidas, bloqueando conexões suspeitas e permitindo apenas comunicações autorizadas. Atua como primeira linha de defesa.
